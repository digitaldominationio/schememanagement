üìã System Overview

The Scheme Monitoring System is a comprehensive platform designed for the Government of Odisha to monitor and manage various welfare schemes efficiently. The system provides end-to-end tracking of financial utilization, beneficiary management, field inspections, and automated reporting.

‚úÖ Complete Implementation: All 4 core modules with 12 functional pages are fully implemented and ready to use.

üéØ Key Objectives

  • Real-time monitoring of scheme implementation across State/District/Block levels
  • Transparent fund allocation, release, and utilization tracking
  • Efficient beneficiary registration and validation with de-duplication
  • Comprehensive field inspection and third-party verification support
  • Automated MIS report generation (Monthly/Quarterly/Annual)
  • PFMS/Treasury integration for financial transparency

‚ùì Frequently Asked Questions

General Questions

Q1: How do I access the system?

A: Open index.html in a modern web browser (Chrome, Firefox, Edge recommended). Enter your credentials provided by the system administrator.

Q2: What browsers are supported?

A: The system works best on modern browsers: Chrome 90+, Firefox 88+, Edge 90+, Safari 14+.

Q3: Can I use this on mobile devices?

A: Yes! The system is fully responsive. The Mobile Inspection Reports module is specifically optimized for field use on smartphones and tablets.

Financial Module

Q4: How do I generate a Utilization Certificate?

A: Navigate to Financial Monitoring ‚Üí UC Generation. Follow the 5-stage workflow: Initiation ‚Üí Documentation ‚Üí Verification ‚Üí Approval ‚Üí Submission.

Q5: What documents are required for UC generation?

A: You need: Expenditure statement, Supporting bills/receipts, Bank statements, Physical progress report, and Photographs of completed work.

Beneficiary Module

Q6: How does the system prevent duplicate registrations?

A: The system uses Aadhaar-based de-duplication. When a beneficiary is registered, the system checks if the Aadhaar number already exists in the database.

Q7: What if a beneficiary doesn't have a bank account?

A: Bank account details are optional during registration but required before benefit disbursement. Beneficiaries can update their details later.

Inspection Module

Q8: Can inspections be done offline?

A: Yes, the Mobile Inspection Reports module supports offline data entry. Reports will sync automatically when internet connectivity is restored.

Q9: How do I upload inspection photos?

A: In the inspection form, use the "Upload Photos/Documents" field. You can capture photos directly using your device camera or upload from gallery.

Reports Module

Q10: Can I schedule automatic report generation?

A: Yes, in the MIS Reports module, you can set up scheduled reports that are automatically generated and emailed to specified recipients.

Q11: What export formats are available?

A: All reports can be exported in PDF, Excel (.xlsx), and CSV formats.

Technical Questions

Q12: Where is the data stored?

A: Currently, the system uses static data for demonstration. For production deployment, integrate with a backend database (MySQL/PostgreSQL recommended).

Q13: How do I integrate with PFMS?

A: PFMS integration requires API credentials from PFMS. Contact your system administrator to configure the API endpoints and authentication.

Q14: Can I customize the district/block/panchayat lists?

A: Yes, these are configurable. The current implementation includes all 30 Odisha districts with sample blocks and panchayats.
